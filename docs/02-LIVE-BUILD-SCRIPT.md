# Live Build Script

**What to say while Claude runs. Keep it loose — this is narration, not a speech.**

---

## Build #1 — Pickleball Site

### Before typing the prompt

> "Watch this. I'm going to speak a prompt — Whispr Flow types it for me. Then hit enter."

*[Dictate or paste Prompt 1. Don't rush — let the audience see what's going in.]*

> "That's it. Hit enter."

*[Hit enter. Step back slightly.]*

---

### While Claude builds (60–90 seconds)

**Pick 2–3 of these lines — don't say all of them. Let silence fill the gaps:**

> "While this runs — notice what's happening. I described what I wanted. Claude is now writing every line of HTML and CSS to make it real. Structure, sections, content — all of it."

*[pause 5–10 seconds — let them watch]*

> "I didn't draw the layout. I didn't write any code. I just described it."

*[pause — watch the output scrolling]*

> "This is vibe coding. You describe what you want. The AI figures out how."

*[if it's still building:]*
> "Almost there..."

---

### When it finishes — open the preview

*[open the HTML file in browser — don't say anything for 3–4 seconds]*

*[let them look — it's intentionally plain, that's the point]*

> "There it is. Working website. Games, leaderboard, sign-up form."

*[3-second pause]*

> "Basic. No personality. No brand. That's on purpose — because watch what happens next."

---

## Build #2 — Plan Mode + Reaction Game

### Before enabling Plan Mode

> "Now I'm going to show you something different. This is called Plan Mode."
>
> "Plan Mode tells Claude: before you write anything, show me the plan. Think it through first. So instead of just building, it maps out the whole problem."

*[Enable Plan Mode — shift+tab in Claude Code]*

> "I'm going to ask Claude to invent a brand for this site, redesign everything, and add a game. One prompt. But it plans first."

*[Dictate or paste Prompt 2. Hit enter.]*

---

### While Plan Mode runs (30–60 seconds of plan output)

> "Watch what it's doing. It's not building yet. It's deciding."

*[as the plan appears, point out 2–3 specific lines — look for branding decisions and game logic:]*

> "It's choosing the brand identity first — name, colors, visual direction. Then it's planning the game logic — the random delay, the timing, the scoring. It's doing in 30 seconds what a product team would spend a day on."

*[pause]*

> "This is how good builders work. You think before you build. Plan Mode makes that the default."

---

### Show the plan before approving

*[scroll up slightly to show the full plan output]*

> "So here's the plan Claude is proposing. [Read one or two items]. Does this look right to me? Yes. Let's approve it."

*[Approve / continue — Claude now builds]*

> "Now it builds."

---

### While Build #2 runs (60–90 seconds)

*[less narration needed here — audience has seen the pattern]*

> "Same as before. But this time, everything it's writing was planned first. Less chaos. More control."

*[pause, let it build]*

---

### When it finishes — open the preview

*[open / refresh the browser — land at the top of the page, don't scroll yet]*

*[pause 3–4 seconds — let the transformation register visually]*

> "Same site."

*[another pause]*

> "Claude invented the name, chose the color palette, built the hero, restyled every section. And —"

*[scroll down to the game]*

> "— added a working game. Let's try it."

*[click Play — react visibly to the ball, click it quickly]*

> "287 milliseconds. Elite, apparently. I'll take it."

*[if the audience laughs or reacts, hold there — don't rush]*

> "That's a branded product. From a plain HTML page. One prompt, six minutes."

---

## Emergency Lines

**If something breaks mid-build:**
> "This is the reality of live demos — things happen. This is actually a good teaching moment: when it breaks, you just describe the problem back to Claude and it fixes it. That's the workflow."

**If Claude produces something ugly or wrong:**
> "This happens. This is Prompt 1 of this feature — and Prompt 1 is never the final version. That's the process: you iterate."

**If Plan Mode takes too long / is verbose:**
> "It's being thorough — which is actually what you want. In real use you can scroll through this or ask it to summarize."
